/datum/listener_tick_cacheListener tick caches are responsible for storing the "listeners by type" lists calculated by PassToChannel() for a single tick. This allows all other hearing operations occuring on the same tick from the same origin to quickly retrieve the relevant listeners; this is why speaking into over ten radios at once doesn't grind the game to a halt.
